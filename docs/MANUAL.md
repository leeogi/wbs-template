## 1. はじめに

WBS アプリは、プロジェクトのタスクを **一覧表（WBS テーブル）** と **ガントチャート** で管理するツールです。

### 特徴

- **リアルタイム同期** — 誰かがタスクを更新すると、他の人の画面にも自動で反映されます
- **ブラウザだけで操作可能** — インストール不要。PC・タブレットのブラウザからアクセスできます
- **元に戻す（Undo）対応** — 操作を間違えても `Cmd+Z`（Mac）/ `Ctrl+Z`（Windows）で元に戻せます
- **AI 更新** — 議事録などから AI を使ってタスクを一括更新できます

---

## 2. アクセスとログイン

### アクセス方法

ブラウザで WBS アプリの URL にアクセスしてください。

### ログイン

アクセスするとパスワード入力画面が表示されます。

- チームで共有しているパスワードを入力して「Enter」を押してください
- ログイン状態はブラウザのタブを閉じるまで保持されます
- 次回アクセス時は再度パスワード入力が必要です

## 3. 画面の見かた

画面は **左右2つのエリア** に分かれています。

```
┌─────────────────────────────┬───────────────────────────────┐
│         ツールバー           │          ツールバー            │
├─────────────────────────────┼───────────────────────────────┤
│                             │                               │
│    WBS テーブル（左側）      │    ガントチャート（右側）      │
│    タスクの一覧・編集        │    スケジュールの可視化        │
│                             │                               │
└─────────────────────────────┴───────────────────────────────┘
```

- **左右の境界線をドラッグ** すると、テーブルとガントの表示比率を変えられます
- テーブルとガントチャートは **スクロールが連動** しています

### ツールバー（画面上部）

| ボタン / 項目 | 説明 |
|:---|:---|
| 担当者フィルター | 特定の担当者のタスクだけ表示 |
| ステータスフィルター | 特定のステータスのタスクだけ表示 |
| 遅延のみ | チェックすると、期限超過タスクだけ表示 |
| タスク追加 | 新しいタスクを追加 |
| フェーズ追加 | 新しいフェーズ（大区分）を追加 |
| カテゴリ追加 | 新しいカテゴリ（中区分）を追加 |
| Export JSON | 現在のタスクデータをファイルとして保存 |
| Import JSON | ファイルからタスクデータを読み込み |
| ↩ / ↪ | 元に戻す（Undo）/ やり直す（Redo） |
| 開始日 / 終了日 | ガントチャートの表示期間を変更 |

## 4. タスクの手動操作（基本操作）

### 4.1 タスクの階層構造

タスクは以下の5段階で管理されています。

| レベル | 名前 | 例 | 説明 |
|:---|:---|:---|:---|
| 1 | フェーズ | 設計フェーズ | 最も大きなまとまり |
| 2 | カテゴリ | UI設計 | フェーズ内の中区分 |
| 3 | エピック | トップページ作成 | カテゴリ内のまとまり |
| 4 | タスク | ワイヤーフレーム作成 | 実際の作業単位 |
| 5 | マイルストーン | デザインレビュー完了 | 重要な節目 |

### 4.2 タスクの編集

テーブルの各セルをクリックすると、その場で編集できます。

| 列 | 操作方法 |
|:---|:---|
| **タスク名** | クリックして文字を入力 |
| **担当者** | クリックしてドロップダウンから選択。複数人の場合はカンマ区切り。名前の横の「×」で個別削除 |
| **ステータス** | クリックしてドロップダウンから選択（未着手 / 対応中 / 確認中 / 完了 / 保留） |
| **リンク** | 関連ドキュメントの URL を入力。🔗 アイコンクリックでリンク先を新しいタブで開く |
| **開始日** | `M/D` 形式で入力（例: `3/15`）。年は自動補完されます |
| **終了日** | 同上 |

> **ポイント**: 編集内容は自動保存されます。「保存」ボタンを押す必要はありません。

### 4.3 タスクの追加

**方法 1: ツールバーから追加**

- 「タスク追加」→ 一番下にタスクが追加される
- 「フェーズ追加」→ 一番下にフェーズが追加される
- 「カテゴリ追加」→ 一番下にカテゴリが追加される

**方法 2: 右クリックメニューから追加**

1. 追加したい位置の行を **右クリック**
2. メニューから選択:
   - 「タスクを下に追加」
   - 「エピックを下に追加」
   - 「カテゴリを下に追加」

### 4.4 タスクの削除

1. 削除したい行を **右クリック**
2. 「削除」を選択

> **注意**: フェーズやカテゴリを削除すると、その配下のタスクもすべて削除されます。

### 4.5 タスクの並び替え

タスクの行を **ドラッグ＆ドロップ** で移動できます。

1. 行の左端あたりを掴む
2. 移動先までドラッグ
3. ドロップして確定

### 4.6 タスクの昇格・降格

タスクの階層レベルを変更できます。

1. 対象の行を **右クリック**
2. 「昇格」→ 1つ上の階層に変更（例: タスク → エピック）
3. 「降格」→ 1つ下の階層に変更（例: エピック → タスク）

### 4.7 折りたたみ・展開

フェーズやカテゴリの行にある **▼ アイコン** をクリックすると、配下のタスクを折りたたんで非表示にできます。もう一度クリックすると展開されます。

### 4.8 元に戻す / やり直す

| 操作 | ショートカット | ツールバー |
|:---|:---|:---|
| 元に戻す（Undo） | `Cmd+Z`（Mac）/ `Ctrl+Z`（Win） | ↩ ボタン |
| やり直す（Redo） | `Cmd+Shift+Z` / `Ctrl+Shift+Z` | ↪ ボタン |

- 最大50回前の操作まで遡れます
- ページをリロードすると履歴はリセットされます

## 5. ガントチャートの使いかた

### 見かた

- **薄い緑のバー**: 予定期間（開始日〜終了日）
- **濃い緑のバー**: 実績期間
- **赤いバー**: 期限超過（終了日を過ぎても完了していない）
- **黄色い縦線**: 今日の日付
- **グレーの背景列**: 土日

### 表示期間の変更

ツールバー右側の「開始日」「終了日」の日付を変更すると、ガントチャートの表示範囲を調整できます。

### 遅延タスクの確認

テーブルの左端に **赤い縦線** が表示されている行は、期限を超過しているタスクです。ツールバーの「遅延のみ」チェックボックスで、これらだけを抽出できます。

## 6. フィルター・絞り込み

ツールバーのフィルターで、表示するタスクを絞り込めます。

| フィルター | 使いかた |
|:---|:---|
| **担当者** | ドロップダウンから名前を選択 → その人が担当するタスクだけ表示 |
| **ステータス** | ドロップダウンからステータスを選択 → そのステータスのタスクだけ表示 |
| **遅延のみ** | チェックを入れる → 期限超過タスクだけ表示 |

フィルターを解除するには、ドロップダウンで空欄（全件表示）を選択してください。

## 7. データのエクスポート・インポート

### エクスポート（バックアップ）

1. ツールバーの **「Export JSON」** をクリック
2. `tasks_YYYY-MM-DD.json` というファイルがダウンロードされます
3. 定期的にエクスポートしておくと、バックアップとして安心です

### インポート（データの復元・差し替え）

1. ツールバーの **「Import JSON」** をクリック
2. 以前エクスポートした JSON ファイルを選択
3. **現在のデータがすべて上書き** されます

> **注意**: インポートは全データを置き換えます。現在のデータが消えるので、事前にエクスポートでバックアップを取ってから実行してください。

## 8. AIによるタスク更新

議事録やミーティングメモの内容をもとに、AI を使ってタスクを一括更新できます。

### 更新の流れ

1. **管理者に依頼する** — 更新したい内容（議事録、進捗メモなど）を共有してください
2. **AIが差分を生成** — AI が内容を分析し、どのタスクをどう更新するか提案します
3. **プレビュー確認** — 更新前に変更内容が一覧表示されるので、内容を確認します
4. **更新を実行** — 確認後、データベースに反映されます
5. **画面に自動反映** — WBS アプリを開いていれば、リアルタイムで更新が反映されます

### 更新できる項目

AI 更新では以下の項目を一括で変更できます:

- ステータス（未着手 → 対応中 など）
- 進捗率（0〜100%）
- 担当者
- 開始日・終了日
- 実績開始日・実績終了日
- 優先度
- メモ

### 具体例

たとえば、定例ミーティングで以下のような話が出た場合:

> 「A1.3のドキュメント作成は完了した。A2.1の設計レビューは田中さんが対応中で進捗50%くらい。」

この内容をもとに、AI が以下のような更新を自動生成します:

| タスクID | 項目 | 変更前 | 変更後 |
|:---|:---|:---|:---|
| A1.3 | ステータス | 対応中 | 完了 |
| A1.3 | 進捗 | 70% | 100% |
| A2.1 | ステータス | 未着手 | 対応中 |
| A2.1 | 進捗 | 0% | 50% |
| A2.1 | 担当者 | — | 田中 |

## 9. ステータス・優先度の定義

### ステータス一覧

| ステータス | 色 | 意味 |
|:---|:---|:---|
| 未着手 | 🔴 赤 | まだ作業を開始していない |
| 対応中 | 🟢 緑 | 作業を進めている |
| 確認中 | 🟡 黄 | 作業は終わり、レビュー・確認待ち |
| 完了 | 🔵 青 | 完全に終了した |
| 保留 | ⚪ グレー | 何らかの理由で一時停止中 |

### 優先度一覧

| 優先度 | 色 | 意味 |
|:---|:---|:---|
| 高 | 🔴 赤 | 最優先で対応が必要 |
| 中 | 🟡 黄 | 通常の優先度 |
| 低 | 🟢 緑 | 急ぎではない |

## 10. よくある質問（FAQ）

### Q. 間違えてタスクを消してしまった

**A.** すぐに `Cmd+Z`（Mac）/ `Ctrl+Z`（Windows）を押してください。元に戻せます。ページをリロードしてしまった場合は、直前に Export したバックアップからインポートで復元してください。

### Q. 他の人の変更が反映されない

**A.** 通常はリアルタイムで反映されますが、反映されない場合はブラウザをリロードしてください。

### Q. ガントチャートにバーが表示されない

**A.** そのタスクに「開始日」と「終了日」の両方が設定されているか確認してください。片方だけではバーは表示されません。また、ガントチャートの表示期間（ツールバー右側の日付）がタスクの日付を含んでいるか確認してください。

### Q. タスクの ID（A1.3 など）は自分で決められる？

**A.** はい。タスク名の左側に表示されている ID はテーブル上で編集できます。ただし、他のタスクと重複しないようにしてください。

### Q. スマートフォンから操作できる？

**A.** ブラウザからアクセスは可能ですが、テーブルの横幅が広いため PC での操作を推奨します。

### Q. データはどこに保存されている？

**A.** Supabase（クラウドデータベース）に保存されています。ブラウザを閉じてもデータは消えません。

## 11. フィードバック・改善要望

機能の追加要望、バグ報告、使いにくい点などがあれば、チームのコミュニケーションツールで管理者に共有してください。

フィードバックには以下を含めると助かります:

- **何をしたかったか**（やりたいこと）
- **何が起きたか**（実際の動作・問題）
- **スクリーンショット**（あれば）
